# hotel-parser — инструкции для Claude

Pet-проект: парсит отели с Booking/Trip/Ostrovok + собирает интерактивную HTML-карту города (отели по ценам, метро, достопримечательности с фото и приоритетами, современные «интересные места» 2025-2026 с категориями) + Excel-отчёт. Сейчас сконфигурирован на Минск.

**Типичная задача:** «сделай такую же карту для города X на даты Y» — алгоритм ниже.

## Архитектура

- **`hotel-parser.js`** — Playwright-парсер трёх источников (не-headless, нужно решать капчи руками). Пишет append-only JSONL-чекпоинт `output/run_<ts>/hotels.jsonl` после каждого отеля + финальный `output/latest.json`.
- **`build-report.js`** — генератор отчёта. Читает `latest.json` + `hotel-config.json`, выдаёт `output/hotels_<slug>_<date>.xlsx` (5 листов) + `_map.html` (Leaflet).
- **`hotel-config.json`** — единственный source of truth для параметров: город/даты/гости + ручные координаты отелей + метро + достопримечательности + интересные места.

## Алгоритм: карта для нового города

### 1. Заполнить базовые поля `hotel-config.json`

Скопируй структуру Минска. Поменяй только:

- `город`: название, страна, `query_booking`, `query_trip`, `cityid_trip` (если знаешь Trip ID — иначе оставь null), `lat`/`lng` центра, `slug` (lowercase ASCII).
- `даты`: ISO `YYYY-MM-DD` для заезда/выезда. По умолчанию строй на 3 ночи; не ограничивайся жёстко — пользователь может хотеть 2 или 5.
- `гости`: взрослых/номеров.

Координаты центра →
```bash
curl -s 'https://nominatim.openstreetmap.org/search?q=<city>&format=json&limit=1' \
  | jq '.[0] | {lat,lon}'
```

### 2. Собрать контент карты — параллельно через research-агенты

Это самая объёмная часть. Делегируй **одновременно** (один Agent-вызов с `run_in_background: true` для каждой темы):

#### a) Метро (если есть в городе)

Запросить у агента: все станции с координатами, номер линии, флаг `пересадка` для пересадочных. Источник: ru/en Wikipedia «Список станций X метрополитена» + кросс-проверка через OSM Nominatim. Точность ±50 м.

Цвета линий, используемые в `build-report.js`:
- Линия 1 → `#CC0000` (красный)
- Линия 2 → `#0055AA` (синий)
- Линия 3 → `#168E44` (зелёный)
- Линия 4+ → потребуется добавить в объект `colors` внутри `buildHtmlMap()` (там два места: polyline + label).

#### b) Ручные координаты топ-отелей

Для 10-15 хорошо известных сетевых отелей города (Hilton, Marriott, Renaissance, Crowne Plaza, Ibis, Holiday Inn, Hyatt, Radisson, Novotel, локальные знаковые отели). **Зачем это нужно:** Nominatim для адресов с номерами домов часто врёт на 0.5-1.5 км. Manual override через substring-match по имени отеля бьёт это.

Координаты — напрямую из OSM по имени:
```bash
curl -s -A 'hotel-parser/1.0' \
  'https://nominatim.openstreetmap.org/search?q=DoubleTree+by+Hilton+<City>&format=json&limit=1&extratags=1' \
  | jq '.[0] | {lat,lon,display_name}'
```

Формат в config (ключ — substring lowercase):
```json
"ручные_координаты": {
  "doubletree": { "lat": 53.9082, "lng": 27.5483 },
  "ренессанс":  { "lat": 53.8921, "lng": 27.5290 }
}
```

Включай и латиницу, и кириллицу, и переводы (Renaissance/Ренессанс) — парсер отдаёт разные варианты с разных сайтов.

#### c) Достопримечательности (5-10 классических)

Поля: `название`, `описание` (1-2 предложения), `lat`, `lng`, `тип` (одиночный emoji), `приоритет` 1-3 (3=топ, 1=если останется время), `фото` (массив URL).

**Фото берём через Wikipedia REST API**, по 1-3 на точку:
```bash
# 1. Получить главное фото статьи:
curl -s 'https://ru.wikipedia.org/api/rest_v1/page/summary/<URL-encoded-title>' \
  | jq '.originalimage.source // .thumbnail.source'

# 2. Если вернулась SVG-схема (бывает для площадей/карт), взять JPG из списка images:
curl -s 'https://ru.wikipedia.org/w/api.php?action=query&titles=<title>&prop=images&imlimit=20&format=json' \
  | jq '.query.pages[].images[].title' | grep -iE '\.(jpg|jpeg|png)$' | head -3

# 3. Конвертить File:Title в актуальный thumb URL:
curl -s 'https://commons.wikimedia.org/w/api.php?action=query&titles=<File:Title>&prop=imageinfo&iiprop=url&iiurlwidth=960&format=json' \
  | jq '.query.pages[].imageinfo[].thumburl'
```

**Не валидируй URL программно** через bulk HEAD — Wikimedia рандомно режет 4xx при пачке HEAD-запросов, выкидывая живые URL. На фронте уже стоит `onerror=placeholder` (серый прямоугольник, высота сохраняется), этого достаточно.

#### d) Интересные места (15-25 современных)

Эта часть — главное отличие от Яндекс.Карт. Делегируй research-агенту с конкретными источниками:
- Локальные тревел-блоги города (для Минска: onliner.by, citydog.io; для Тбилиси: bm.ge, agenda.ge; и т.д.)
- WHERETOEAT/Tripadvisor топы 2025-2026
- Reddit r/<city>
- TikTok/Instagram-вирусные точки

**Категории строго эти 5** (ключи и эмодзи зафиксированы в `PLACE_CATEGORIES` build-report.js):
- `еда` — оранжевый `#FB8C00` — рестораны, кафе, фастфуд (включая локальные сети типа Mak.by)
- `тусовка` — фиолетовый `#8E24AA` — бары, караоке, арт-кварталы, концертные площадки
- `шопинг` — синий `#1E88E5` — ТЦ, маркеты, концептуальные магазины
- `природа` — зелёный `#43A047` — парки, набережные, водоёмы внутри города
- `загород` — коричневый `#6D4C41` — точки в радиусе 1-2 часов на машине (замки, этно-музеи, природа)

Поле `тренд: true` для тех точек, что в топах 2025-2026 — на карте они получают бейдж 🔥 в попапе и легенде.

**Загородные точки обязательны.** Минимум 2-3 за пределами города в радиусе 2ч езды (для Минска — Несвижский/Мирский замки, Дудутки; для Тбилиси — Мцхета, Кахетия, Степанцминда). Они отображаются на карте с авто-уменьшением zoom при клике из легенды.

#### e) Фото для интересных мест (опционально)

Wikipedia/Commons — да для парков/замков/инфраструктуры (надёжно). Для современных ресторанов/баров — сложно, hotlinkable публичных URL обычно нет. Если не нашёл — пиши `"фото": []` (либо опускай поле). Не выдумывай URL.

### 3. Запуск

```bash
npm install                      # один раз
npx playwright install chromium  # один раз

# Изменить hotel-config.json под новый город

npm run parse                    # 5-15 минут, не headless — будут капчи
npm run report                   # < 30 сек
open output/hotels_<slug>_*_map.html
```

`npm run parse:booking` / `parse:trip` / `parse:ostrovok` — запустить только один источник.

## Грабли и принципы

- **Nominatim адреса часто промахиваются** на 500-1500 м даже в больших городах. Поэтому: ручные координаты для known hotels + валидация по метро (см. `geocodeHotels()` в build-report.js: если адрес говорит «X м от метро Y», а Nominatim вернул точку дальше `max(X*2.5, X+400)` от Y — фолбэк на станцию метро).
- **Wikipedia thumbnail URL формат:** только через REST API или MediaWiki API. Выдуманные URL вида `Minsk_Independence_Square.jpg/640px-...` не работают (404). Текущий список фото в config был получен через API — не подменяй на «угаданные».
- **Категории интересных мест** — не вводи новые без обновления `PLACE_CATEGORIES` в build-report.js (цвет, фон, label) И легенды.
- **Загородные точки** — координаты в config используются как есть; в `inCityBounds()` (для отелей) они не проходят, но это проверка только для отелей. Для интересных мест нет проверки bounds.
- **Кликабельная легенда** работает через `markerByName[name] = L.marker(...)`. Имена должны быть уникальны в рамках всех типов маркеров (отели/достопримечательности/места).

## Команды

| Команда | Что делает |
|---|---|
| `npm run parse` | парсер всех трёх источников |
| `npm run parse:booking` | только Booking |
| `npm run parse:trip` | только Trip.com |
| `npm run parse:ostrovok` | только Ostrovok.ru |
| `npm run report` | сгенерировать отчёт (xlsx + html) из последнего `latest.json` |
